The project now uses a flat deployable runtime layout: all QuantConnect algorithm code is self-contained in `qc_runtime/` with exactly ten modules (`main.py`, `config.py`, `features.py`, `regime.py`, `scoring.py`, `sizing.py`, `execution.py`, `risk.py`, `reporting.py`, `universe.py`), while tests remain in `tests/test_all.py` with fixture data under `tests/fixtures/` and CI enforced by `.github/workflows/walk_forward.yml`.
Scalper runtime defaults to cash-account / Canada compliance via `qc_runtime/config.py` `ENABLE_SHORTS = False` (`enable_shorts=False`), which disables the `momentum_short` sleeve unless explicitly enabled.
Default portfolio mode is now `strategy_mode="momentum"` for low-frequency cross-sectional rotation; the scalper sleeve remains implemented and can be enabled explicitly by setting `strategy_mode="scalper"` in `qc_runtime/config.py`.
