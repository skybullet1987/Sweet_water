The project now uses a flat deployable runtime layout: all QuantConnect algorithm code is self-contained in `qc_runtime/` with exactly ten modules (`main.py`, `config.py`, `features.py`, `regime.py`, `scoring.py`, `sizing.py`, `execution.py`, `risk.py`, `reporting.py`, `universe.py`), while tests remain in `tests/test_all.py` with fixture data under `tests/fixtures/` and CI enforced by `.github/workflows/walk_forward.yml`.
